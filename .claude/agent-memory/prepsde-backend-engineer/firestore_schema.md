All collections are top-level (not subcollections). Each document has a `userId` field for scoping queries.

**Collections:**
- `users/{uid}` — profile, streakCount, targetDate, geminiLastCalledAt, currentPhase
- `problems/{problemId}` — DSA problems with SRS fields (reviewCount, nextReviewDate, mastered)
- `reflections/{reflectionId}` — doc ID format: `{userId}_{date}` (e.g., `abc123_2026-06-14`) for O(1) existence check
- `streaks/{uid}` — current/longest streak, lastActiveDate, history map (90-day rolling window)
- `deadlineAdjustments/{adjustmentId}` — each deadline change event with daysDelta and reason
- `weeklySnapshots/{snapshotId}` — doc ID format: `{uid}_{weekStartDate}`, generated by Cloud Scheduler weekly

**Key field names (exact — use these when writing Firestore queries):**
- `userId` (string) — on problems, reflections, deadlineAdjustments, weeklySnapshots
- `nextReviewDate` (string, ISO date "YYYY-MM-DD") — on problems
- `reviewCount` (number 0-3) — on problems; determines SRS interval
- `mastered` (boolean) — on problems
- `aiVerdict` ("deep_work" | "surface" | "lazy" | null) — on reflections
- `geminiLastCalledAt` (ISO datetime string | null) — on users; daily quota gate
- `targetDate` (string, ISO date) — on users; the current adjusted deadline
- `originalDeadline` (string, ISO date) — on users; never changes after onboarding

**SRS intervals (3-7-15 rule, from useDSATracker.ts):**
- reviewCount=0 → nextReviewDate = today + 3 days
- Review complete (not struggled): reviewCount+1, next interval = [3,7,15,30][reviewCount]
- Review where struggled=true: reviewCount → 0, nextReviewDate = today + 3 days
- mastered=true when reviewCount >= 3 and not struggled

**Verdict values:** "deep_work" | "surface" | "lazy" (NOT "productive"/"mediocre" from old PWA)

**weeklySnapshot document ID:** `{uid}_{weekId}` where weekId = ISO week format "YYYY-WNN" (e.g., "abc123_2026-W24"). Generated every Sunday by Cloud Scheduler POST /progress/weekly.

**reflection document ID:** `{userId}_{date}` (e.g., "abc123xyz_2026-06-14"). Enforces one reflection per user per day at the DB layer. Idempotent create via `set(..., merge=True)`.

**deadlineAdjustments idempotency:** Before writing, check `userId == uid AND computedAt >= todayStart`. If exists, skip — prevents double-writes when Cloud Scheduler fires twice.

**pods collection (deferred, not MVP):** Top-level `pods/{podId}` with subcollection `pods/{podId}/members/{uid}`. `memberCount` is denormalized on the pod document. Member stats (thisWeekProblems, thisWeekStreak) are duplicated from user doc for read performance.

**Composite indexes needed (must create in firestore.indexes.json):**
- problems: userId ASC + nextReviewDate ASC
- problems: userId ASC + pattern ASC + nextReviewDate ASC
- problems: userId ASC + pattern ASC + mastered ASC
- problems: userId ASC + mastered ASC + createdAt DESC
- problems: userId ASC + difficulty ASC + nextReviewDate ASC
- reflections: userId ASC + date DESC
- reflections: userId ASC + submittedAt DESC
- deadlineAdjustments: userId ASC + computedAt DESC
- weeklySnapshots: userId ASC + weekStartDate DESC
